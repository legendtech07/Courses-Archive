A — What is an NFT (quick definition)

An NFT (non-fungible token) is a unique crypto token that represents ownership of a specific digital or physical asset. Each NFT is distinct (token ID + contract address). NFTs usually point to metadata (JSON) that describes the asset (image, name, attributes).

B — Standards you must know

  • ERC-721 — classic NFT standard (one token = one item).
  • ERC-1155 — multi-token standard for fungible & non-fungible in one contract (good for games).
  • Metaplex — Solana NFT tooling & metadata standards.
  • EIP-2981 — royalty metadata standard (marketplaces may or may not enforce it).

C — Metadata & storage

NFT metadata is usually a JSON file pointed to by tokenURI. Use IPFS (content-addressed) and pinning services (nft.storage, Pinata) to ensure persistence. Example metadata below follows OpenSea-friendly schema.

{

"name": "Legend #1", "description": "My first NFT", "image": "ipfs://Qm...ImageCID", "external_url": "https://legendtech.example/item/1", "attributes": [ { "trait_type": "Rarity", "value": "Rare" }, { "trait_type": "Color", "value": "Crimson" } ], "animation_url": "ipfs://Qm...OptionalAnimationCID", "license": "CC-BY-4.0" }

D — How minting actually works (practical flow)

  1. Create assets & metadata JSON.
  2. Upload image + JSON to IPFS → get CIDs.
  3. Deploy an ERC-721 / ERC-1155 or use marketplace minting; set tokenURI to ipfs://....
  4. List on marketplaces (OpenSea, Magic Eden) which index your contract.

Tip: use L2s or Polygon for cheap minting; mainnet costs vary with gas.

E — Lazy minting & royalties

Lazy minting defers on-chain minting until purchase, saving creators gas. Royalties (EIP-2981) expose royalty info, but marketplace enforcement varies. Consider technical & business strategies for creator revenue.

F — Use cases & types of NFTs

  • Collectibles / PFPs
  • Utility / Membership NFTs
  • Gaming assets (often ERC-1155)
  • Music / media tokens
  • Fractionalized NFTs & tokenized real-world assets
  • Dynamic NFTs (metadata changes over time)

G — Metaverse notes

Metaverse = persistent 3D virtual spaces where users socialize, play, own land & assets. Examples: Decentraland, The Sandbox. It's early and fragmented; real utility varies by project.

H — Legal / IP / Ethics

Owning an NFT ≠ owning copyright. Clarify licenses in metadata. Permanent links create long-term responsibility — moderate content before minting. Royalties and legal frameworks evolve — consult counsel for complex cases.

I — Security & best practices

  • Use OpenZeppelin implementations.
  • Multisig admin keys (Gnosis Safe).
  • Test metadata immutability & verify CIDs.
  • Plan redundant pinning & backup storage.
