Quick Definition

A PLC (Programmable Logic Controller) is an industrial computer used to control machines, automate processes, and make decisions based on inputs and outputs. Think of a PLC like a super-reliable robot brain sitting inside factories, controlling things like:

  • Elevators
  • Traffic lights
  • Water pumps
  • Bottle-filling machines
  • Conveyor belts
  • Robots
  • Power stations

A PLC takes INPUTS ‚Üí makes a DECISION ‚Üí sends OUTPUTS.

Every PLC works in a loop (scan cycle):

  1. Read Inputs ‚Äî Sensors, switches, buttons, temperature probes
  2. Execute Program ‚Äî Your ladder logic or structured text
  3. Update Outputs ‚Äî Motors, lights, relays, actuators
  4. Repeat very fast ‚Äî Like 1‚Äì10 milliseconds

This cycle is called the SCAN CYCLE ‚Äî the soul of a PLC.

(Simulation: Inputs ‚Üí Execute ‚Üí Outputs ‚Üí Repeat)

1. Digital Inputs & Outputs

Digital = ON/OFF. Examples: push button ‚Üí Input; motor ‚Üí Output. If input is ON ‚Üí output can be ON.

2. Ladder Logic Basics (LD)

3. Latching / Holding Circuits

Used to keep something ON even after removing the button.

// Start ---( )--- Motor

//             | //           Motor (This latches the motor on)

4. Timers (TON, TOFF, TP)

TON = Timer On Delay. Example: Turn on a fan 5 seconds after pressing a button.

5. Counters (CTU, CTD)

Count things: bottles, cars, people, pulses. Example: start alarm after 10 items.

6. Internal Bits / Flags

Markers used inside PLC like variables: M0.0, B3:0/2, I/O memory.

7. Analog Inputs (0‚Äì10V / 4‚Äì20mA)

Used for temperature, pressure, level sensors. Convert analog values into engineering scaled values.
